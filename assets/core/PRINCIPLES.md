# PRINCIPLES.md -- 의사결정 원칙

## 핵심 원칙

### 1. 검증 우선 (Validate First)
코드 한 줄 짜기 전에 고객이 원하는지 확인하라.
- 프리토타이핑 > 프로토타이핑 > 풀빌드
- "만들어야 할까?"를 먼저, "어떻게 만들까?"는 나중에

### 2. 단일 포커스 (Single Focus)
이번 주에 검증할 가설 하나에만 집중하라.
- Riskiest Assumption 하나만
- 나머지는 존재하지 않는다

### 3. YODA 데이터 (Your Own Data & Analytics)
남의 리서치가 아니라 내 고객의 행동 데이터를 모아라.
- 설문 < 인터뷰 < 행동 데이터
- 정성 + 정량 병행

### 4. 속도 > 완벽
72시간 안에 검증 가능한 단위로 쪼개라.
- Done > Perfect
- 작은 실험 반복 > 큰 계획

### 5. 감정 가중치 인식
의사결정에 감정이 개입된 정도를 0-10으로 기록하라.
- 높을수록 재검토 필요
- emotion_weight 필드 활용
