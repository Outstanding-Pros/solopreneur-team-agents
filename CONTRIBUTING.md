# Contributing

Thank you for contributing!

---

## Areas of Contribution

### 1. Adding/Improving Agents

Create or modify `agents/{team}/{agent}/SKILL.md` files.

**Required SKILL.md Sections:**
```markdown
# [Agent Name]

> One-line description

## Trigger
- When this agent gets activated

## Input
- Required/optional input items

## Process
- Step-by-step processing logic

## Output
- Artifact format

## Quality Checklist
- Quality validation criteria

## Handoff
- What to pass to the next step
```

When adding a new agent, also add keyword mappings to `AGENT_ROUTES` in `src/bot.py`:
```python
AGENT_ROUTES = {
    "keyword": ("team", "agent-slug"),
    ...
}
```

### 2. Adding/Modifying Routines

Create or modify `routines/{routine-id}.md` files.

Routines follow this structure:
```markdown
# Routine Name

Description

## Files to Read
- @./product/brief.md
- @./memory/experiments.jsonl

## Output Format
(Specific format definition)
```

When adding a new routine:
1. Register it in the `ROUTINES` list in `src/scheduler.py`
2. Also register it in the `ROUTINES` list in `cli/run_routine.py`
3. Specify `memory_targets` if auto-saving to JSONL memory is needed

### 3. CLI Tools

Add new CLI tools to the `cli/` directory.

Rules:
- Use `rich` library for terminal UI
- Load product list from `core/products.json`
- Determine project paths via `REPOS_BASE_PATH` environment variable
- Handle CLI arguments with `argparse`

### 4. PRD Templates

Create `templates/prd-{type}.md` files.

Template variables use `{{variable_name}}` format.

### 5. Documentation

Add design docs, guides, etc. to the `docs/` directory.

## Code Style

### Python

- Use Python 3.10+ syntax
- Async code: `asyncio` + `async/await`
- Claude Code invocation: `claude --print` (pass prompt via stdin)
- Config loading: `python-dotenv` + `.env` file
- Product info: `core/products.json` (auto-generated by setup.py)

### SKILL.md Writing Principles

1. **Clear triggers**: Be specific about when activation occurs
2. **Structured process**: Define each step clearly
3. **Actionable artifacts**: Output should be immediately usable
4. **Quality checklist**: Enable self-validation

### Naming Conventions

- Agents: `lowercase-with-hyphen` (e.g., `user-researcher`)
- Routines: `lowercase-with-hyphen.md` (e.g., `signal-scan.md`)
- CLI: `lowercase_with_underscore.py` (e.g., `run_routine.py`)
- Templates: `prd-{type}.md` (e.g., `prd-pmf.md`)

## Pull Request Guide

### PR Title Format
```
[Agent] Add branding agent
[Routine] Add competitor-watch routine
[CLI] Add export command
[Bot] Improve agent routing accuracy
[Fix] Fix scheduler memory append
[Docs] Update installation guide
```

### PR Description Should Include
- Reason for the change
- Summary of changes
- How to test (if applicable)
