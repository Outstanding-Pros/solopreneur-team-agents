# Solo Founder Agents 설치 가이드

> 개발 경험이 없어도 따라할 수 있는 설치 안내서

---

## 이 시스템이 뭔가요?

혼자서 제품을 만드는 1인 창업자를 위한 **AI 비서 시스템**입니다.

스타트업에는 전략팀, 마케팅팀, 디자인팀, 개발팀이 필요합니다. 하지만 1인 창업자에게는 이 모든 역할을 혼자 해야 하는 현실이 있습니다. 이 시스템은 **AI가 그 팀원들을 대신**합니다. 설치 한 번이면 24시간 쉬지 않는 AI 팀이 당신 옆에서 함께 일합니다.

---

## 왜 이 시스템인가?

### 1. 23명의 전문 AI 팀원이 생긴다

혼자 일하지만 혼자가 아닙니다. 이 시스템에는 4개 팀, 23명의 AI 전문가가 내장되어 있습니다.

| 팀 | 하는 일 | 소속 에이전트 |
|----|--------|-------------|
| **전략팀** | 시장 분석, 사업 기획, 가설 수립, 일정 산정 | PMF Planner, Feature Planner, Data Analyst, Business Strategist, Idea Refiner, Scope Estimator, Policy Architect |
| **그로스팀** | 마케팅 전략, 콘텐츠 작성, 브랜딩, 광고 | GTM Strategist, Content Writer, Brand Marketer, Paid Marketer |
| **경험팀** | 유저 리서치, 시장 조사, UX/UI 설계 | User Researcher, Desk Researcher, UX Designer, UI Designer |
| **엔지니어링팀** | 프론트/백엔드 개발, API, 데이터, 인프라 | Creative Frontend, FDE, Architect, Backend Developer, API Developer, Data Collector, Data Engineer, Cloud Admin |

Discord에 "랜딩페이지 카피 써줘"라고 보내면 Content Writer가, "경쟁사 분석해줘"라고 보내면 Desk Researcher가 자동으로 배정됩니다. 누구를 부를지 고민할 필요 없이 **그냥 하고 싶은 말을 하면** 알아서 적절한 전문가가 답합니다.

### 2. 24시간 자동으로 일한다

잠든 사이에도, 다른 일을 하는 동안에도, AI가 알아서 일합니다.

| 시간 | AI가 하는 일 | 왜 필요한지 |
|------|------------|-----------|
| **아침 6시** | 오늘의 브리핑 | 일어나면 바로 오늘 뭘 해야 할지 정리되어 있음 |
| **낮 12시** | 시장 시그널 탐지 | 경쟁사 변화, 고객 트렌드, 기술 변화를 놓치지 않음 |
| **오후 4시** | 실험 상태 점검 | 진행 중인 실험이 잘 되고 있는지 자동 체크 |
| **밤 10시** | 하루 기록 | 오늘 뭘 했고 내일 뭘 이어갈지 자동 정리 |
| **일요일 저녁** | 주간 회고 | 한 주를 돌아보고 다음 주 계획 자동 제안 |

이 결과는 모두 **Discord 채널에 자동 보고**됩니다. 아침에 일어나서 Discord만 열면 오늘 할 일이 정리되어 있습니다.

### 3. 내가 한 모든 결정과 실험이 쌓인다

대부분의 AI 도구는 대화가 끝나면 맥락이 사라집니다. 이 시스템은 다릅니다.

- **가설 관리**: 어떤 가설을 세웠고, 지금 어떤 상태인지 추적
- **실험 기록**: 어떤 실험을 했고, 결과가 어땠는지 자동 기록
- **의사결정 로그**: 언제, 왜 그런 결정을 했는지 근거와 함께 저장
- **시그널 아카이브**: 시장에서 포착한 변화들을 날짜별로 축적

이 기록들은 시간이 지날수록 가치가 올라갑니다. AI가 과거 데이터를 참고해서 더 정확한 분석과 제안을 하게 됩니다. "지난달에 실험한 거 결과가 어땠지?"라고 물으면 바로 찾아서 알려줍니다.

### 4. 제품이 여러 개여도 섞이지 않는다

1인 창업자는 동시에 여러 제품을 시도하는 경우가 많습니다. 이 시스템은 **3단계 격리 구조**로 제품 간 맥락이 절대 섞이지 않습니다.

```
[공통 레이어]  오너의 일하는 방식, 의사결정 원칙, 글쓰기 스타일
     ↓
[제품 레이어]  제품별 브리프, 타깃 고객, 가설, 실험 기록
     ↓
[프로젝트 레이어]  프로젝트별 목표, 에이전트 배정, 진행 상태
```

쇼핑몰 제품의 AI는 SaaS 제품의 데이터를 볼 수 없습니다. 각 제품마다 완전히 독립된 AI 팀이 운영됩니다.

설치 시 제품을 여러 개 등록하기만 하면, 각각 자동으로 전용 폴더 + 전용 메모리 + 전용 Discord 채널이 만들어집니다.

### 5. 설치 한 번이면 끝이다

복잡한 설정이 필요 없습니다. 설치 마법사가 질문을 하고, 당신은 답만 하면 됩니다.

- 이름이 뭔가요?
- 어떤 제품을 만들고 있나요?
- Discord 봇 토큰은?

이것만 입력하면 **제품 폴더 생성, AI 메모리 초기화, Discord 채널 세팅, Docker 빌드**까지 전부 자동입니다.

### 6. 검증 중심 철학이 내장되어 있다

이 시스템의 AI들은 단순히 "시키는 대로" 하지 않습니다. 1인 창업자가 가장 빠지기 쉬운 함정 — **검증 없이 만드는 것** — 을 막아주는 원칙이 내장되어 있습니다.

- **검증 우선**: "만들어야 할까?"를 "어떻게 만들까?"보다 먼저 질문
- **단일 포커스**: 이번 주에 검증할 가장 위험한 가설 하나에만 집중
- **72시간 단위**: 모든 실험을 72시간 안에 결과를 볼 수 있는 크기로 분해
- **감정 가중치**: 의사결정에 감정이 얼마나 개입됐는지 기록해서, 나중에 재검토

AI가 "이거 먼저 검증하는 게 좋겠다", "이 기능은 아직 만들 단계가 아니다"라고 말하는 이유는 이 원칙 때문입니다.

## 설치 전 준비물

총 3가지가 필요합니다. 하나씩 안내합니다.

### 준비물 1: Claude Code Max 구독

AI를 실행하는 엔진입니다. 월 구독 서비스입니다.

1. https://claude.ai 접속
2. 로그인 (없으면 회원가입)
3. 좌측 메뉴 또는 설정에서 **Claude Code** 항목 찾기
4. **Max 플랜** 구독

> 이 시스템은 Claude Code라는 터미널 도구를 사용합니다.
> Max 플랜을 구독하면 사용량 제한 없이 쓸 수 있습니다.

### 준비물 2: Docker Desktop

이 시스템을 24시간 자동으로 돌려주는 프로그램입니다.

1. https://www.docker.com/products/docker-desktop/ 접속
2. 본인 운영체제(Windows / Mac / Linux)에 맞는 버전 다운로드
3. 설치 후 실행 → 화면 아래 트레이에 고래 아이콘이 뜨면 성공

> Docker는 "프로그램을 담는 상자"라고 생각하면 됩니다.
> 한번 설치해두면 이 시스템이 알아서 그 안에서 돌아갑니다.

### 준비물 3: Discord 봇 토큰

AI가 Discord에서 메시지를 주고받으려면 "봇 계정"이 필요합니다.

1. https://discord.com/developers/applications 접속 (Discord 로그인 필요)
2. 우측 상단 **New Application** 클릭
3. 이름 입력 (예: "내 AI 비서") → **Create**
4. 왼쪽 메뉴에서 **Bot** 클릭
5. **Reset Token** 클릭 → **Yes, do it!**
6. 나타난 토큰을 **복사해서 메모장에 저장** (한 번만 보여줌!)

> 이 토큰은 비밀번호와 같습니다. 절대 다른 사람에게 공유하지 마세요.

같은 페이지에서 추가 설정:
- **Bot** 페이지 아래로 스크롤
- **Privileged Gateway Intents** 섹션에서 3개 모두 켜기:
  - Presence Intent → ON
  - Server Members Intent → ON
  - Message Content Intent → ON
- **Save Changes** 클릭

## 설치 (약 10분)

### Step 1: 파일 다운로드

**터미널**(또는 명령 프롬프트)을 엽니다.

- **Mac**: Spotlight(Cmd+Space)에서 "터미널" 검색
- **Windows**: 시작 메뉴에서 "명령 프롬프트" 또는 "PowerShell" 검색

아래 명령어를 **그대로 복사해서 붙여넣기** 후 Enter:

```
git clone https://github.com/seungw1n/solo-founder-agents.git
```

```
cd solo-founder-agents
```

> `git`이 없다는 오류가 나오면:
> - Mac: 터미널에 `xcode-select --install` 입력 후 Enter
> - Windows: https://git-scm.com 에서 설치

### Step 2: 설치 마법사 실행

```
python setup.py
```

> `python`이 없다는 오류가 나오면:
> - https://www.python.org 에서 최신 버전 설치
> - 설치 시 **"Add to PATH" 체크박스를 반드시 선택**

설치 마법사가 시작되면, 화면의 질문에 답하면 됩니다:

```
── 1단계: 환경 확인 ──
✓ Docker
✓ Python 3.10+
✓ git

── 2단계: 기본 설정 ──
이름: (본인 이름 입력)
직군/역할: (예: 기획자, 디자이너, 개발자 등)
Discord Bot Token: (아까 복사해둔 토큰 붙여넣기)
프로젝트 저장 경로: (그냥 Enter 치면 기본값 사용)

── 3단계: 제품/조직 등록 ──
제품/조직 이름: (관리할 제품 이름 입력, 예: 내쇼핑몰)
제품을 더 추가할까요?: (여러 개면 y, 아니면 n)

── 4단계: Docker 빌드 ──
(자동으로 진행됩니다. 2~3분 걸림)

── 5단계: Discord Bot 초대 ──
(화면에 나오는 안내를 따라가세요)

── 6단계: 시스템 시작 ──
(자동)
```

### Step 3: Discord 봇 초대

5단계에서 설치 마법사가 **초대 링크**를 보여줍니다.

1. 그 링크를 브라우저에 붙여넣기
2. 봇을 추가할 Discord 서버 선택
3. **승인** 클릭

> Discord 서버가 없으면:
> Discord 앱 좌측 하단 **+** 버튼 → 서버 만들기

### Step 4: Claude Code 로그인 (마지막!)

Docker 안에서 Claude Code에 로그인해야 합니다. 한 번만 하면 됩니다.

```
docker compose exec bot claude login
```

화면의 안내를 따라 Claude 계정으로 로그인하면 끝입니다.

## 설치 확인

Discord 서버에 가보면 AI가 자동으로 만든 채널들이 보입니다:
- `#owner-command` — 여기에 메시지를 보내면 AI가 답함
- `#daily-brief` — 매일 아침/저녁 자동 브리핑
- `#signals` — 시장 시그널 자동 탐지
- `#experiments` — 실험 상태 자동 점검
- `#weekly-review` — 매주 일요일 자동 회고

**테스트**: `#owner-command` 채널에 아무 메시지나 보내보세요. AI가 답하면 성공!

## 일상적인 사용법

### Discord에서 AI에게 일 시키기

`#owner-command` 채널에 자연어로 메시지를 보내면 됩니다.

```
랜딩페이지 카피 초안 써줘
→ AI가 content-writer 역할로 응답

우리 경쟁사 분석해줘
→ AI가 desk-researcher 역할로 응답

이번 주 실험 결과 정리해줘
→ AI가 data-analyst 역할로 응답
```

메시지 내용에 따라 23개 전문 에이전트 중 적절한 역할이 자동으로 선택됩니다.

### 자동 루틴 (손 안 대도 됨)

매일 자동으로 실행됩니다:

| 시간 | 무엇을 하는지 | 어디에 보고 |
|------|-------------|------------|
| 아침 6시 | 오늘 할 일 정리 | #daily-brief |
| 낮 12시 | 경쟁사/시장 변화 탐지 | #signals |
| 오후 4시 | 실험 진행 상태 점검 | #experiments |
| 밤 10시 | 오늘 한 일 기록 | #daily-brief |
| 일요일 저녁 8시 | 한 주 돌아보기 | #weekly-review |

### 새 프로젝트 만들기

터미널에서:
```
python cli/new_project.py
```

화면의 질문에 답하면 프로젝트 폴더가 자동 생성됩니다.

### 전체 현황 보기

```
python cli/status.py
```

### 수동으로 루틴 실행하기

자동 시간까지 기다리지 않고 바로 실행하고 싶을 때:
```
python cli/run_routine.py
```

## 제품을 여러 개 운영할 때

설치 시 제품을 여러 개 등록하면, 각각 완전히 분리된 AI 공간이 만들어집니다.

```
예시:
- 제품 A: 쇼핑몰 → AI가 쇼핑몰 맥락만 보고 일함
- 제품 B: SaaS → AI가 SaaS 맥락만 보고 일함
```

제품 A의 AI는 제품 B의 정보를 절대 보지 못합니다. 섞이지 않습니다.

나중에 제품을 추가하려면 `python setup.py`를 다시 실행하면 됩니다.

## 자주 묻는 질문

**Q: 비용이 얼마나 드나요?**
A: Claude Code Max 구독료만 있으면 됩니다. 나머지는 무료입니다.

**Q: 컴퓨터를 꺼도 돌아가나요?**
A: Docker가 실행 중이면 백그라운드에서 계속 돌아갑니다. 컴퓨터를 꺼면 멈추고, 다시 켜면 자동 시작됩니다.

**Q: 시간대가 한국이 아니면?**
A: `docker-compose.yml` 파일을 텍스트 편집기로 열고 `TZ=Asia/Seoul`을 본인 시간대로 바꾸면 됩니다. (예: `TZ=America/New_York`)

**Q: AI가 틀린 답을 하면?**
A: AI는 도구입니다. 중요한 결정은 반드시 본인이 직접 확인하세요. AI의 분석을 참고 자료로 활용하는 것을 권장합니다.

**Q: 시스템을 멈추고 싶으면?**
A: 터미널에서 `docker compose down` 입력.

**Q: 다시 시작하려면?**
A: 터미널에서 `docker compose up -d` 입력.

**Q: 업데이트하려면?**
A: 터미널에서 순서대로:
```
git pull
docker compose build
docker compose up -d
```

## 문제가 생겼을 때

### "python을 찾을 수 없습니다"
→ Python이 설치되지 않았거나 PATH에 등록되지 않은 상태입니다.
→ https://www.python.org 에서 재설치하되, **"Add to PATH" 체크박스를 반드시 선택**하세요.

### "docker를 찾을 수 없습니다"
→ Docker Desktop이 실행 중인지 확인하세요. (화면 하단 트레이에 고래 아이콘)
→ 없으면 https://www.docker.com/products/docker-desktop/ 에서 설치.

### "DISCORD_TOKEN 없음"
→ `.env` 파일에 Discord Bot Token이 입력되지 않은 상태입니다.
→ `python setup.py`를 다시 실행하면 됩니다.

### Discord에서 봇이 응답하지 않음
→ `docker compose logs bot` 으로 오류 확인
→ 봇이 서버에 초대되었는지 확인
→ `#owner-command` 채널에서 메시지를 보냈는지 확인 (다른 채널에서는 응답하지 않음)

### 루틴이 실행되지 않음
→ `docker compose logs scheduler` 로 오류 확인
→ `docker compose restart scheduler` 로 재시작
